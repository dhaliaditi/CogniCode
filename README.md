# ⚙ CogniCode

**Predicting human-perceived Python code quality using static analysis and machine learning.**

CogniCode predicts how human expert developers would rate Python code across four dimensions: Complexity, Readability, Maintainability, and Refactoring Necessity. It extracts 63 static analysis metrics from your code and feeds them into trained machine learning models that approximate human expert judgment.

Built on research at the University of Saskatchewan. Trained on ratings from 35 professional Python developers across 401 files from 11 open-source AI projects (TensorFlow, PyTorch, scikit-learn, and others).

---

## Quality Dimensions

| Dimension | Model | Performance | Reliability |
|---|---|---|---|
| Complexity | Random Forest | κ = 0.52, 281% of individual reviewer | ✅ Trustworthy |
| Refactoring | Random Forest | AUC = 0.76, 99% of individual reviewer | ✅ Reliable screening |
| Readability | SVM RBF | κ = 0.32, 34% of consensus ceiling | ⚠️ Guide only |
| Maintainability | SVM RBF | κ = 0.36, 42% of consensus ceiling | ⚠️ Guide only |

Scores are on a 1–4 Likert scale (higher = better quality). Refactoring shows probability (0–100%).

## Setup

**Requirements:** Python 3.10+, git

### Step 1 — Clone the repository

```bash
git clone https://github.com/YOUR_USERNAME/cognicode.git
cd cognicode
```

### Step 2 — Run setup

```bash
bash setup.sh
```

`setup.sh` does the following automatically:
- Creates a Python virtual environment (`venv/`) inside the project folder
- Installs all dependencies with pinned versions into that venv (numpy 1.24.4, scikit-learn 1.3.2, Flask, Radon, Pylint, Prospector, and others)
- Checks whether the pre-built models are compatible with your environment
- Creates a `run.sh` shortcut

The venv keeps everything isolated — nothing is installed system-wide and it will not interfere with your existing Python setup.

### Step 3 — Start the tool

```bash
./run.sh
```

Open **http://localhost:5050** in your browser.

## Usage

Three input modes are available in the web UI:

- **Local Path** — type the path to a `.py` file or project directory on your machine
- **Upload .py** — upload a single Python file directly from your browser
- **GitHub URL** — paste a public GitHub repo URL (requires outbound internet access)

## Static Analysis Tools

CogniCode uses five static analysis tools to extract 51 code metrics:

| Tool | What it measures | Requires |
|---|---|---|
| Radon | Cyclomatic complexity, Halstead metrics, Maintainability Index | pip (included) |
| Pylint | Style violations, coding standard issues | pip (included) |
| Prospector | Aggregated linting signals | pip (included) |
| SonarQube | Code smells, cognitive complexity, technical debt, security | Docker |
| Understand | Architecture, call structure, deep coupling metrics | Licence + `und` binary |

Radon, Pylint, and Prospector run automatically. SonarQube and Understand are optional but improve prediction accuracy, especially for Readability and Maintainability.

### Enabling SonarQube

Start SonarQube via Docker:

```bash
docker-compose up -d
```

Then pass your SonarQube details when running:

```bash
python3 app.py --sonar-host http://localhost:9000 --sonar-user admin --sonar-password admin
```

### Enabling Understand

Pass the path to your `und` binary:

```bash
python3 app.py --understand-bin /path/to/und
```

## Research Background

This tool is the practical output of the paper:

> *CogniCode: Predicting Human-Perceived Code Quality using Static Analysis and Machine Learning*
> Journal 
Key findings:
- No single static metric replicates human judgment on any dimension
- Linear combinations overfit and fail to generalize
- Machine learning bridges the gap partially, with a clear replication spectrum
- Complexity is structurally measurable; Readability and Maintainability depend on semantic signals that current metrics cannot capture
- Three metrics appear in the top-10 for all four dimensions: `comment_lines_density`, `statements`, and `ClassName` — from three different tools, validating the multi-tool approach

## Disclaimer

CogniCode scores approximate human expert judgment. They are not a replacement for code review. Complexity predictions are most reliable. Readability and Maintainability scores should be used alongside, not instead of, human review.
